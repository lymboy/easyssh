# EasySSH v2.0 设计文档

## 概述

EasySSH 是一个终端 SSH 服务器管理工具，本次优化目标是：
1. 美化 CLI 输出，提升用户体验
2. 改善错误处理，移除不友好的堆栈打印
3. 修复现有 bug
4. 重写 README.md 提升项目吸引力

---

## 1. 视觉风格：专业企业风

### 1.1 服务器列表展示

采用紧凑表格格式，类似 kubectl 风格：

```
NAME                  GROUP        HOST              USER      STATUS
─────────────────────────────────────────────────────────────────────
web-master           prod         192.168.1.10     root      ● Ready
web-slave           prod         192.168.1.11     root      ● Ready
db-primary          prod         192.168.1.20     admin     ○ Idle
─────────────────────────────────────────────────────────────────────
dev-server          dev          10.0.0.5         dev       ○ Idle
test-server         dev          10.0.0.6         dev       ○ Idle
─────────────────────────────────────────────────────────────────────
Total: 5 servers | Production: 3 | Development: 2
```

**设计要点：**
- 表头使用灰色，与数据区分
- root 用户红色高亮显示（安全警示）
- 分组分隔线增强可读性
- 底部统计行：总数 + 各分组数量
- 状态指示器：绿色 ● 在线/已连接，灰色 ○ 空闲

### 1.2 实现方案

修改 `config/config.go` 中的 `PrintServerV2()` 方法：

1. 简化表格列：移除密码列（敏感信息），保留核心信息
2. 添加分组分隔线逻辑
3. 添加底部统计输出
4. 优化颜色方案

## 2. 错误处理：友好提示

### 2.1 设计原则

- **零堆栈打印**：移除所有 `panic`、`log.Fatalf` 调用
- **相似建议**：输入错误时提供相似名称建议
- **统一格式**：错误信息有清晰的视觉边界

### 2.2 错误提示格式

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ✗ 找不到服务器: 'web-production'
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  您是否想连接以下服务器？
  • web-master (root@192.168.1.10)
  • web-slave  (root@192.168.1.11)

使用 'easyssh server ls' 查看完整列表
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 2.3 需要修改的文件

| 文件 | 位置 | 原代码 | 改为 |
|------|------|--------|------|
| `ssh/ssh.go` | L50, L56 | `log.Fatalf(...)` | 返回错误，由上层处理 |
| `ssh/ssh.go` | L135, L149 | `panic(err)` | 返回错误 |
| `ssh/ssh.go` | L184 | `getPassword()` 调用 | 添加错误提示 |
| `cmd/server/server.go` | L34-38 | 直接调用 ssh | 添加错误检查和提示 |

### 2.4 相似名称建议算法

使用 Levenshtein 编辑距离算法：
- 编辑距离 <= 2 视为相似
- 最多显示 3 个建议

## 4. Bug 修复

### 4.1 Keep-Alive 逻辑问题

**问题位置：** `ssh/ssh.go:119` 和 `ssh/ssh.go:176`

**问题描述：**
- 第 119 行创建了 `ticker` 但未使用
- 第 176 行又创建了同名 `ticker`，覆盖了前者
- 保活间隔设置为 5 秒过短，可能导致服务器日志刷屏

**修复方案：**
1. 移除第 119 行无用的 ticker
2. 将保活间隔调整为配置项，默认 60 秒
3. 添加配置项 `keep_alive_interval`

### 4.2 root.go 描述错误

**问题位置：** `cmd/root.go:15-19`

**原代码：**
```go
Use:   "cobra-cli",
Short: "A generator for Cobra based Applications",
```

**改为：**
```go
Use:   "easyssh",
Short: "SSH server management tool for terminal users",
```
